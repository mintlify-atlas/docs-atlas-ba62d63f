## Overview

The Cloro de Hidalgo Procurement System follows Next.js 16 App Router conventions with a clean, modular architecture.

## Root Directory Structure

```
App_Compras/
├── app/                  # Next.js 16 App Router
├── components/           # Reusable React components

├── scripts/              # Utility scripts
├── middleware.ts         # Next.js middleware (auth)

├── next.config.ts        # Next.js configuration
├── package.json          # Dependencies and scripts
└── .gitignore            # Git ignore rules
```

### Component Patterns

#### Form Components
Use React Hook Form + Zod validation:

```typescript

    defaultValues: {...}
```

## Configuration Files

### `middleware.ts`
Next.js middleware for authentication:

```typescript
import { updateSession } from '@/lib/supabase/middleware'

export async function middleware(request: NextRequest) {
    return await updateSession(request)
}

export const config = {
    matcher: ['/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg)$).*)'],
}
```

- Protects `/dashboard/*` routes
- Redirects unauthenticated users to `/login`
- Refreshes Supabase session cookies

## Import Patterns

### Path Aliases

Use `@/` prefix for all imports:

```typescript
import { createClient } from '@/lib/supabase/server'
import { Button } from '@/components/ui/button'
import type { Requisicion } from '@/types'
import { requisicionSchema } from '@/lib/validations/schemas'
```

### Server vs Client Imports

**Server Components/Actions**:
```typescript
import { createClient } from '@/lib/supabase/server'  // Server-side
import { cookies } from 'next/headers'                 // Next.js server
```

**Client Components**:
```typescript
'use client'
import { createClient } from '@/lib/supabase/client'  // Browser-side
import { useState, useEffect } from 'react'
```

## File Naming Conventions

- **Components**: PascalCase (e.g., `RequisicionForm.tsx`)
- **Pages**: lowercase (e.g., `page.tsx`)
- **Actions**: camelCase (e.g., `requisiciones.ts`)
- **Types**: PascalCase interfaces (e.g., `Requisicion`)
- **Utilities**: camelCase (e.g., `utils.ts`)

## Data Flow Architecture

```
User Interaction
    ↓
Client Component (useState, forms)
    ↓
Server Action ('use server' functions)
    ↓
Supabase Client (server.ts)
    ↓
PostgreSQL Database (with RLS)
    ↓
Response to Client
    ↓
UI Update (revalidatePath, redirect)
```
