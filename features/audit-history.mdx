## Overview

The audit history system automatically tracks all changes made to requisitions, recording what changed, who made the change, and when it occurred. This provides complete accountability and traceability for compliance and operational review.

<Info>
Audit records are created automatically whenever a requisition is updated. Users cannot manually create or modify audit history entries.
</Info>

## Data Structure

```typescript types/index.ts
export interface RequisicionHistorial {
    id: string                  // Unique history record ID
    requisicion_id: string      // References parent requisition
    campo_modificado: string    // Field name that changed
    valor_anterior: string | null   // Previous value
    valor_nuevo: string | null      // New value
    usuario_id: string          // User who made the change
    created_at: string          // Timestamp of change

}
```

## Database Schema

```sql supabase/schema.sql
CREATE TABLE requisiciones_historial (
  id                UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  requisicion_id    UUID NOT NULL REFERENCES requisiciones(id) ON DELETE CASCADE,
  campo_modificado  TEXT NOT NULL,
  valor_anterior    TEXT,
  valor_nuevo       TEXT,
  usuario_id        UUID NOT NULL REFERENCES auth.users(id),
  created_at        TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_historial_requisicion ON requisiciones_historial(requisicion_id);
```

<Note>
The `ON DELETE CASCADE` constraint ensures that when a requisition is deleted, all its audit history records are automatically deleted as well.
</Note>

## Automatic Audit Trail Creation

Audit records are created automatically during requisition updates:

```typescript lib/actions/requisiciones.ts
export async function updateRequisicion(
    id: string,
    data: Partial<RequisicionFormData>,
    camposModificados: Array<{ campo: string; anterior: string; nuevo: string }>
) {
    const supabase = await createClient()
    const profile = await getCurrentProfile()

    if (!profile || !['admin', 'coordinadora'].includes(profile.rol)) {
        return { error: 'No tienes permisos para editar requisiciones' }
    }

    const {
        data: { user },
    } = await supabase.auth.getUser()
    if (!user) return { error: 'Usuario no autenticado' }

    // Update the requisition
    const { data: result, error } = await supabase
        .from('requisiciones')
        .update(data)
        .eq('id', id)
        .select()
        .single()

    if (error) return { error: error.message }

    // Write audit trail for all modified fields
    if (camposModificados.length > 0) {
        await supabase.from('requisiciones_historial').insert(
            camposModificados.map((c) => ({
                requisicion_id: id,
                campo_modificado: c.campo,
                valor_anterior: c.anterior,
                valor_nuevo: c.nuevo,
                usuario_id: user.id,
            }))
        )
    }

    revalidatePath('/dashboard/calendar')
    revalidatePath('/dashboard/requisiciones')
    return { data: result }
}
```

## Querying Audit History

### For a Specific Requisition

```typescript lib/actions/requisiciones.ts
export async function getHistorial(requisicionId?: string) {
    const supabase = await createClient()

    let query = supabase
        .from('requisiciones_historial')
        .select(`
            *,
            profiles(id, nombre_completo, rol)
        `)
        .order('created_at', { ascending: false })

    if (requisicionId) {
        query = query.eq('requisicion_id', requisicionId)
    }

    const { data, error } = await query
    if (error) return { error: error.message }
    return { data: data ?? [] }
}
```

### All History Records

Omit the `requisicionId` parameter to fetch all audit records across all requisitions:

```typescript
const { data: allHistory } = await getHistorial()
```

<Note>
History records are ordered by `created_at` descending, so the most recent changes appear first.
</Note>

## User Information

Audit records include user profile information via SQL join:

```sql
SELECT 
  h.*,
  p.id,
  p.nombre_completo,
  p.rol
FROM requisiciones_historial h
LEFT JOIN profiles p ON h.usuario_id = p.id
WHERE h.requisicion_id = $1
ORDER BY h.created_at DESC
```

This provides:
- User's full name
- User's role (admin, coordinadora, etc.)
- Link back to user profile if needed

## Compliance and Auditing

### Tamper-Proof Design

<Warning>
Audit history records are immutable once created. The system provides no interface or API to modify or delete individual history records.
</Warning>

This ensures:
- Complete audit trail integrity
- Compliance with regulatory requirements
- Protection against retroactive manipulation
- Reliable forensic analysis

### Data Retention

Audit records are retained as long as their parent requisition exists:

- **Active requisitions**: Full history available
- **Deleted requisitions**: History deleted via CASCADE
- **No automatic purging**: Records never expire

<Note>
If long-term audit retention is required even after requisition deletion, implement a separate archival process before deletion.
</Note>

## Row-Level Security

Audit history respects the same RLS policies as requisitions:

```sql supabase/schema.sql
ALTER TABLE requisiciones_historial ENABLE ROW LEVEL SECURITY;

CREATE POLICY "historial: authenticated can view" ON requisiciones_historial
  FOR SELECT USING (auth.role() = 'authenticated');
```

All authenticated users can view audit history, but only the system can create records (through server actions).

## Performance Considerations

### Indexed Queries

The `requisicion_id` column is indexed for fast lookups:

```sql
CREATE INDEX idx_historial_requisicion ON requisiciones_historial(requisicion_id);
```

This optimizes queries like:
```sql
SELECT * FROM requisiciones_historial WHERE requisicion_id = 'uuid';
```

### Batch Inserts

Multiple field changes in a single update are inserted in one batch operation:

```typescript
await supabase.from('requisiciones_historial').insert(
    camposModificados.map((c) => ({
        requisicion_id: id,
        campo_modificado: c.campo,
        valor_anterior: c.anterior,
        valor_nuevo: c.nuevo,
        usuario_id: user.id,
    }))
)
```

This reduces database round-trips when updating multiple fields.

## Reporting and Analytics

Audit history enables various analytics:

### Most Frequently Changed Fields

```sql
SELECT campo_modificado, COUNT(*) as changes
FROM requisiciones_historial
GROUP BY campo_modificado
ORDER BY changes DESC;
```

### User Activity Report

```sql
SELECT 
  p.nombre_completo,
  COUNT(*) as total_changes,
  COUNT(DISTINCT h.requisicion_id) as requisiciones_affected
FROM requisiciones_historial h
JOIN profiles p ON h.usuario_id = p.id
GROUP BY p.nombre_completo
ORDER BY total_changes DESC;
```

### Status Change Timeline

```sql
SELECT 
  campo_modificado,
  valor_anterior,
  valor_nuevo,
  created_at
FROM requisiciones_historial
WHERE campo_modificado = 'Estatus'
ORDER BY created_at DESC;
```
