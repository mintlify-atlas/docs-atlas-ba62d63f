## Overview

The Calendar View provides a visual, time-based interface for managing delivery schedules. Built with FullCalendar, it displays requisitions as color-coded events based on their confirmed delivery dates and status.

<Frame>
  <img src="/images/calendar-preview.png" alt="Calendar view showing scheduled deliveries" />
</Frame>

<Info>
Requisitions appear on the calendar based on their **fecha_confirmada** (confirmed date). If no confirmed date is set, the system falls back to **fecha_recepcion** (reception date).
</Info>

## FullCalendar Integration

The calendar uses FullCalendar React with Spanish localization and multiple view options:

```typescript components/calendar/CalendarView.tsx
import FullCalendar from '@fullcalendar/react'
import dayGridPlugin from '@fullcalendar/daygrid'
import timeGridPlugin from '@fullcalendar/timegrid'
import interactionPlugin from '@fullcalendar/interaction'
import esLocale from '@fullcalendar/core/locales/es'

<FullCalendar
    plugins={[dayGridPlugin, timeGridPlugin, interactionPlugin]}
    initialView="dayGridMonth"
    locales={[esLocale]}
    locale="es"
    headerToolbar={{
        left: 'prev,next today',
        center: 'title',
        right: 'dayGridMonth,timeGridWeek',
    }}
    events={events as any}
    eventContent={renderEventContent}
    eventClick={(info) => {
        onEventClick(info.event.extendedProps.requisicion)
    }}
    dayMaxEvents={3}
    height="100%"
    contentHeight="auto"
    aspectRatio={1.5}
    buttonText={{
        today: 'Hoy',
        month: 'Mes',
        week: 'Semana',
    }}
/>
```

## Event Data Transformation

Requisitions are transformed into FullCalendar events with custom styling:

```typescript components/calendar/CalendarView.tsx
const events: CalendarEvent[] = requisiciones.map(req => {
    const eventColor = req.estatus?.color_hex || '#4266ac'
    const title = req.producto?.nombre || 'S/P'
    return {
        id: req.id,
        title,
        start: req.fecha_confirmada || req.fecha_recepcion, // YYYY-MM-DD format
        backgroundColor: eventColor,
        borderColor: eventColor,
        extendedProps: {
            requisicion: req,
            proveedor_nombre: req.proveedor?.nombre || 'N/A',
            estatus_nombre: req.estatus?.nombre || 'N/A',
            estatus_color: eventColor,
        }
    }
})
```

### CalendarEvent Type

```typescript types/index.ts
export interface CalendarEvent {
    id: string
    title: string
    start: string
    backgroundColor: string
    borderColor: string
    extendedProps: {
        requisicion: Requisicion
        proveedor_nombre: string
        estatus_nombre: string
        estatus_color: string
    }
}
```

<Note>
The `extendedProps` object stores the full requisition data, making it available when users click on events.
</Note>

## Custom Event Rendering

Events are rendered with custom UI including status icons and supplier information:

```typescript components/calendar/CalendarView.tsx
const STATUS_ICONS: Record<string, LucideIcon> = {
    'cancelado': XCircle,
    'confirmado': CheckCircle2,
    'en revisión': Search,
    'en tránsito': Truck,
    'pendiente': Clock,
    'recibido': PackageCheck,
}

const renderEventContent = (eventInfo: any) => {
    const { event } = eventInfo
    const props = event.extendedProps
    const color = props.estatus_color || '#4266ac'

    // Convert hex to RGB for background transparency
    const hexToRgb = (hex: string) => {
        const result = /^#?([a-f\d]{2})([a-f\d]{2})([a-f\d]{2})$/i.exec(hex)
        return result ? `${parseInt(result[1], 16)}, ${parseInt(result[2], 16)}, ${parseInt(result[3], 16)}` : '14, 12, 155'
    }

    const rgb = hexToRgb(color)
    const StatusIcon = getStatusIcon(props.estatus_nombre)

    return (
        <div
            className="flex flex-col w-full overflow-hidden rounded-[5px] cursor-pointer"
            style={{
                background: `rgba(${rgb}, 0.10)`,
                borderLeft: `3px solid ${color}`,
                padding: '3px 6px',
            }}
            title={`${event.title} · ${props.proveedor_nombre} (${props.estatus_nombre})`}
        >
            <div className="flex items-center gap-1 min-w-0">
                <StatusIcon
                    className="shrink-0"
                    style={{ width: '10px', height: '10px', color: color }}
                />
                <div
                    className="font-semibold leading-tight truncate"
                    style={{ fontSize: '0.7rem', color: color, letterSpacing: '-0.01em' }}
                >
                    {event.title}
                </div>
            </div>
            <div
                className="truncate leading-snug"
                style={{ fontSize: '0.63rem', color: '#64748b', marginTop: '1px', paddingLeft: '11px' }}
            >
                {props.proveedor_nombre}
            </div>
        </div>
    )
}
```

The modal displays:

## Sidebar Filters

The calendar includes sidebar filters for refined views:

```typescript app/dashboard/calendar/page.tsx
const [filters, setFilters] = useState<RequisicionFilters>({})

const loadData = async () => {
    setLoading(true)
    const { data, error } = await getRequisiciones(filters)
    if (!error && data) {
        setRequisiciones(data as Requisicion[])
    }
    setLoading(false)
}

useEffect(() => {
    loadData()
}, [filters])
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Event not appearing on calendar">
    Verify the requisition has a **fecha_confirmada** set. If null, it uses **fecha_recepcion** as fallback.
  </Accordion>

  <Accordion title="Wrong event color">
    Check the status catalog to ensure the correct color_hex value is assigned to the status.
  </Accordion>

  <Accordion title="Calendar not loading">
    Ensure FullCalendar plugins are installed: `@fullcalendar/react`, `@fullcalendar/daygrid`, `@fullcalendar/timegrid`, `@fullcalendar/interaction`
  </Accordion>
</AccordionGroup>
