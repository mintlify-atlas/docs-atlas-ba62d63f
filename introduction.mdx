# Cloro de Hidalgo Procurement System

The **Cloro de Hidalgo Procurement System** is a comprehensive Next.js 16 + Supabase SaaS application designed to streamline the purchase requisition management workflow for Cloro de Hidalgo S.A. de C.V.

## What is it?

This system provides a centralized platform for managing the complete lifecycle of material procurement, from initial requisition to final delivery. Built with modern web technologies, it offers real-time collaboration, role-based access control, and calendar-based delivery tracking.

## Core Capabilities

### Purchase Requisition Management
- Create and edit requisitions with detailed product, supplier, and delivery information
- Track multiple date fields: reception date, OC date, requested delivery, confirmed delivery, and actual delivery
- Manage quantities, units, invoice/remission numbers, and comments
- Real-time status updates with color-coded visual indicators

### User Management
- Self-service registration with admin approval workflow
- Pending user queue for access requests
- Dynamic role assignment and management
- User profile configuration

### Calendar & Reporting
- Interactive FullCalendar integration showing confirmed delivery dates
- Filter by supplier, destination, status, and date ranges
- Sidebar with upcoming deliveries
- Quick-view event details with inline editing

### Data Architecture

The system uses **Supabase** for:
- PostgreSQL database for relational data storage
- Row Level Security (RLS) for data access control
- Real-time subscriptions for live updates
- Built-in authentication with email/password

## User Roles

The system implements four primary roles with distinct permissions:

| Role | Permissions | Use Case |
|------|-------------|----------|
| **Admin** | Full access: create, edit, delete requisitions; manage users; assign roles | System administrators |
| **Coordinadora** | Create, edit, delete requisitions; manage catalogs; view all data | Procurement coordinators |
| **Laboratorio** | View-only access to requisitions and calendar | Laboratory team members |
| **CEDIS** | View-only access to requisitions and calendar | Distribution center staff |

<Note>
  New users register with a **pendiente** (pending) role and must be approved by an admin or coordinadora before gaining system access.
</Note>
