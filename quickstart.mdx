# Quickstart Guide

This guide will walk you through the complete workflow: from signing up for an account to creating your first purchase requisition in the Cloro de Hidalgo Procurement System.

<Note>
  **Prerequisites**: You need a valid company email address to register. All new accounts require approval from an administrator before gaining full access.
</Note>

## Step 1: Request Access

Navigate to the login page and click on the **"Solicitar Acceso"** (Request Access) tab.

<Steps>
  <Step title="Fill out the registration form">
    Provide your full name, company email, and create a password:
    
    ```typescript
    // Registration form fields
    {
      nombre_completo: "Your Full Name",
      email: "you@clorodehidalgo.com",
      password: "******",
      confirm_password: "******"
    }
    ```
    
    <Note>
      Password must be at least 6 characters long.
    </Note>
  </Step>
  
  <Step title="Submit your request">
    Click **"Solicitar Acceso"** to submit your registration. The system uses Supabase authentication:
    
    ```typescript
    // From lib/actions/auth.ts:23-42
    export async function signUp(formData: { 
      email: string; 
      password: string; 
      nombre_completo: string 
    }) {
      const supabase = await createClient()
      
      const { error } = await supabase.auth.signUp({
        email: formData.email,
        password: formData.password,
        options: {
          data: {
            nombre_completo: formData.nombre_completo,
          }
        }
      })
      
      if (error) {
        return { error: error.message }
      }
      
      revalidatePath('/', 'layout')
      redirect('/dashboard/pendiente')
    }
    ```
  </Step>
  
  <Step title="Wait for approval">
    After successful registration, you'll see a confirmation message:
    
    > ¡Solicitud enviada! Un administrador revisará tu cuenta y te asignará acceso. Recibirás confirmación pronto.
    
    Your account is created with a **pendiente** (pending) role. An administrator must approve your account and assign you a proper role before you can access the system.
  </Step>
</Steps>

## Step 2: Login After Approval

Once your account is approved, you'll receive notification. Return to the login page:

<Steps>
  <Step title="Enter your credentials">
    Use the email and password you registered with:
    
    ```typescript
    // From lib/actions/auth.ts:7-21
    export async function signIn(formData: { email: string; password: string }) {
      const supabase = await createClient()
      
      const { error } = await supabase.auth.signInWithPassword({
        email: formData.email,
        password: formData.password,
      })
      
      if (error) {
        return { error: error.message }
      }
      
      revalidatePath('/', 'layout')
      redirect('/dashboard/calendar')
    }
    ```
  </Step>
  
  <Step title="Access the dashboard">
    After successful login, you'll be redirected to `/dashboard/calendar` - the main calendar view.
  </Step>
</Steps>

<Warning>
  If you see an error like "Invalid login credentials", double-check your email and password. If you're still unable to login, contact your administrator to verify your account was approved.
</Warning>

## Step 4: Create Your First Requisition

<Note>
  Only users with **admin** or **coordinadora** roles can create requisitions. If you don't see the "Nueva Requisición" option, you have a view-only role.
</Note>

<Steps>
  <Step title="Navigate to Nueva Requisición">
    Click **"Nueva Requisición"** in the sidebar, or go to `/dashboard/requisiciones/nueva`
  </Step>
  
  <Step title="Fill in basic information">
    The form is organized into several sections:
    
    **Section 1: Requisition Data (Manual Entry)**
    - № REQUI - Requisition number (e.g., "F0000")
    - FECHA DE OC - Purchase order date
    - FOLIO OC - Purchase order folio (e.g., "GG-01000")
    - FECHA SOLICITADA - Requested delivery date
    - FECHA CONFIRMADA - Confirmed delivery date (required, appears on calendar)
    - FACTURA / REMISIÓN - Invoice or remission number
    
    <Warning>
      **FECHA CONFIRMADA** is required and can only be edited by admin or coordinadora roles. This is the date that appears on the calendar.
    </Warning>
  </Step>
  
  <Step title="Select catalog items">
    **Section 2: Material Details (from Catalogs)**
    
    Choose from existing catalogs or use the **+** button to quickly add new items:
    
    - **PROVEEDOR** (Supplier) - Required
    - **PRODUCTO** (Product) - Required  
    - **PRESENTACIÓN** (Presentation) - Required
    - **DESTINO** (Destination) - Required
    - **ESTATUS** (Status) - Required
    - **CANTIDAD Y UNIDAD** (Quantity and Unit) - Required
    
    ```typescript
    // Quick add functionality from components/forms/RequisicionForm.tsx:322-326
    <Button

    ```
    
    The **+** button opens a quick-add modal to create catalog entries on-the-fly without leaving the form.
  </Step>
  
  <Step title="Add delivery information (optional)">
    **Section 3: Delivery Information**
    
    - FECHA ENTREGADO - Actual delivery date (filled when received)
    - CANTIDAD ENTREGADA - Actual quantity delivered
  </Step>
  
  <Step title="Add comments (optional)">
    Use the **COMENTARIOS / NOTAS** field for any additional instructions or notes about the requisition.
  </Step>
  
  <Step title="Submit the form">
    Click **"Generar Nueva Requisición"** to create the requisition:
    

  </Step>
</Steps>

## Common Workflows

### Creating a Requisition from Receipt

1. Navigate to **Nueva Requisición**
2. Enter the reception date (fecha_recepcion)
3. Select supplier, product, and destination from catalogs
4. Enter requested quantity and unit
5. Leave delivery dates empty initially
6. Submit to create

### Updating Confirmed Delivery Date

1. Find the requisition in calendar or table view
2. Click to open detail modal
3. Click **Edit** button
4. Update **FECHA CONFIRMADA** field (admin/coordinadora only)
5. Save changes - the calendar will update automatically

### Marking a Requisition as Delivered

1. Open the requisition detail
2. Edit the requisition
3. Fill in **FECHA ENTREGADO** (actual delivery date)
4. Enter **CANTIDAD ENTREGADA** (actual quantity received)
5. Add **FACTURA / REMISIÓN** number if applicable
6. Update status to "Entregado" or similar
7. Save changes

<Note>
  All changes are automatically tracked in the requisition history (historial) table for audit purposes.
</Note>

## Troubleshooting

**Can't create requisitions?**
- Verify your role is admin or coordinadora
- Check with your administrator if you need elevated permissions

**Requisition not appearing on calendar?**
- Ensure the **FECHA CONFIRMADA** field is filled in
- Check that your filters aren't hiding the event

**Error when submitting form?**
- Verify all required fields (marked with *) are filled
- Check that you've selected valid catalog entries
- Ensure dates are in the correct format
